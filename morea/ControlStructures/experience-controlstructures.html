You are expected to do your own work on all homework assignments. You may (and are encouraged to) engage in general discussions with your classmates regarding the assignments, but specific details of a solution, including the solution itself, must always be your own work. How to turn in?

Assignments need to be turned in via Laulima. Check the Syllabus for the late assignment policy for the course.

What to turn in?

You must turn in a zip archive named ics312_hw5_xxx.zip, where “xxx” is your UH user name (e.g., for me, it would be ics312_hw5_henric.zip). The archive must contain a single top-level directory called ics312_hw5_xxx, where “xxx” is your UH user name (e.g., for me, it would be ics312_hw5_henric). In that directory you must have all the files named exactly as specified in the questions below.

Expected contents of the ics312_hw5_xxx directory:

  • hw5_ex1.asm: code for Exercise #1
  • asm_io.asm: I/O functions source code
  • asm_io.inc: I/O include file
  • driver.c: C driver
  • Makefile: A makefile that compiles your programs into an executable (we must just be able to type “make” and generate your executables). We will only grade the executable generated by typing “make”.

Deviation from the above specs will lead to lost points:

  • Archive is not a zip archive (-5 pts)
  • Archive does not extract as a single directory (-5 pts)
  • The name of the directory is incorrect (-5 pts)
  • The asm_io.asm file is missing (-5 pts)
  • The asm_io.inc file is missing (-5 pts)
  • The driver.c file is missing (-5 pts)
  • The archive is empty, or doesn’t extract, or does not contain a makefile: no credits for the assignment

Exercise 1: Measuring English-ness of a corpus [90 pts]

Overview

In this exercise we develop a program that takes as input a string terminated by a newline character. That string contains text written in some (natural) language, and the ultimate goal is to write a program that makes some statement about how likely the text is to be in English. Obviously this is not an AI course, and given how difficult assembly programming is we can only implement something pretty basic.

Question #1 [40 pts]: Inputing a (large) string and printing character counts

Write an assembly program called hw5_ex1, stored in file hw5_ex1.asm, which prompts the user for a string. The program then prints the count for each of the 26 letters of the alphabet, not distinguishing between upper case and lower case, and ignoring all non-letter characters.

Implementation Constraint:

  • You cannot declare more than 10 labels in your program (counting .data and .bss segment labels together). If you do you’ll receive no credit for the assignment.

Hints:

  • Since you don’t know how long the string will be, there is no way you can store it in RAM. Instead, you should do all processing on-the-fly, i.e., as each character comes in;
  • Clearly it’s a good idea to use an array of 26 counters (the first counter is for counting occurrences of ‘a’, the second counter is for counting occurrences of ‘b’, etc.);
  • We’ll run your code for very large input string (see questions below), so these counters should store 4-byte values to avoid overflow.

Question #3 [20 pts]: Determining English-ness

A very coarse way to automatically determine the language of a corpus is to compare character occurrences. By processing a large corpus in a language one can sort the characters by decreasing frequencies, as done by our program above. Several researchers have done this and a proposed order for English is: “etaoinshrdlcumwfgypbvkjxqz” (credited to Robert Lewand’s book Cryptological Mathematics). So this means that in English the most frequent letter is ‘e’, the second most frequent is ‘t’, etc.

Enhance your program so that it prints an English-ness score: the length of the longest prefix match between the letters sorted by your program and the English order. For instance, if your program says that the letters sorted by decreasing numbers of occurrences are: “etaoishnrdlcumwfgypbvkjxqz”; then the English-ness score would be 5 (i.e., both orders have the same five first letters e, t, a, o, and i, but the sixth differs).
